## Body

People should be able to discover by convention whether a description document exists, such as `/openapi.yaml` or just use `/openapi` and provide YAML by default and allow content-negotiation if you prefer another format.

Might also be wise to finally register media types, like `application/openapi+json` and `application/openapi+yaml`?

Media type registration for OpenAPI and JSON Schema is under way https://ietf-wg-httpapi.github.io/mediatypes/draft-ietf-httpapi-rest-api-mediatypes.html  
Media type registration for YAML is currently in working group last call. 
https://ietf-wg-httpapi.github.io/mediatypes/draft-ietf-httpapi-yaml-mediatypes.html

If we create a well-known identifier identifier it is worth considering using the IETF syntax for well known resources https://www.rfc-editor.org/rfc/rfc8615
