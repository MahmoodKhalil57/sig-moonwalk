### Comment 1 — @earth2marsh · 2025-06-10 16:52Z

Topic: Moonwalk has been making progress in two notable ways...

1. Imagining bold, new futures of the spec
2. Recognizing what value might actually be realized within the 3.x branches (signatures, ADA/DOM)

The latter is very much in the spirit of what we used to call OpenAPI.Next. What if we were to lean more into that? Keeping in mind where we could take the next major version, why not bring as much of the value to 3.x as we can now? That would help realize improvements in the ecosystem sooner. Then when we're ready to roll out things like syntax changes to enable signatures we cannot support in 3.x (e.g., multi-path segments), the leap to 4.0 should be much easier to make (and in keeping with the mechanical upgrade principle).

So the suggestion is to reframe how we think about the work we've done and will do in Moonwalk as a way to balance these two goals of defining how we see the new world we're headed toward while improving the one we have today.
