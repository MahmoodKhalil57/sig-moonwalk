### Comment 1 — @handrews · 2023-12-07 16:53Z

OAS 3.1 already uses "actual" JSON Schema 2020-12 (with a few extensions that for validation can be safely ignored, but can impact code gen, e.g. `discriminator`).

I cannot imagine Moonwalk going backwards on this, but as the recent blog post about shipping Moonwalk in 2024 notes, thre is also a desire to loosen the coupling between OAS and any specific schema system (which I agree with as a goal).  My guess is that it wil retain the ability to use "actual" JSON Schema (whatever that means in 2024- it's [not entirely clear](https://github.com/orgs/json-schema-org/discussions/557) where JSON Schema is headed these days), but may support alternatives.  But that's just a guess on my part.
