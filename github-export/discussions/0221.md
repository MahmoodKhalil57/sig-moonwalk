### Comment 1 — @handrews · 2026-02-09 17:28Z

I'd like to talk about the scope of our specifications.

The OAS is primarily about mapping data between HTTP serialized forms and application-suitable forms.  It also includes some meta-data, the most notable of which are tags, which can be used to add semantics, but are a low-level construct without inherent meaning (you have to register specific meanings).

LLMs care much more about _application-level_ semantics.  We are also getting more and more requests for features that I feel are more application-level rather than HTTP serialization-level, such as sentinel events in event streams, data sensitivity levels, or API lifecycle semantics.

Note also that formalizing this could go with a slight rebrand of "OpenAPI Specification" (vague and confusing now that we have three specifications) to "OpenAPI Serialization".  And potentially make "Moonwalk" an "OpenAPI Semantics" specification.
