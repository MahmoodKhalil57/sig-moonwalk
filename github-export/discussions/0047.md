  > 
  > * Import TypeSpec: Another TypeSpec file you've built your model on.
  > * Import JavaScript: Some way you've extended TypeSpec, which you'll use in your model.
  > 
  > In essence, nothing ends up in your Python program - you are only importing to your TypeSpec model. You'd use an emitter to generate Python from the model that you "could" use in your implementation, if you chose to work that way.
